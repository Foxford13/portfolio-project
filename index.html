My Projects

That was a first project we had to do at General Assembly that we were asked to do after only two weeks of learning how to code. The premise was simple. We come up with a game, then we code in Javascript. I decided to make Othello/Reversi board game in which each player has to flank enemies playing pieces in order to have fill out the board with as many pieces of his own.
One of the biggest challenges was to create a system of recoginising coordinates of different pieces so the players wouldn’t be allowed to make an invalid move. The solution to this problem was to calculate the distance between each piece by taking into account the index number of each square, and applying a set of rules to the code that would allow only legal move.

JavaScript, jQuery, HTML and CSS.

Our second project required us to create a RESTful up with at least one API hooked up, and a login and register form for the user. I focused on creating a database for book exchange points where people could exchange books freely between each other. The two APIs I used were Google Books and Google Maps. Their main purpose was to improve user experience and help locate where a given book may be, by giving a search option for to book that the user wants to upload, and by giving it's location on the map.

HTML5, SASS, JavaScript ES6, JQuery 3.10, Gulp, Yarn, Git, GitHub, MongoDB, Mongoose, Express, Amazon Web Services (S3 Bucket).

For the last project I decided to work on my own in order to test my abilities.  After being introduced to Ruby and Rails we were required to create an application that implemented them in the backend, and AngularJS at the front. Purpose of my application was to create a place where people could buy and sell second hand computer parts. One of the main features of my app is in-messaging system so the prospective buyers could contact the seller. The logic was setup at the back end in Ruby. This was particularly challenging as I couldn’t allow to have multiple conversation between only two people. This problem was solved by setting up appropriate validation forms as well as setting up of the controllers.
